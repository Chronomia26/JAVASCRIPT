What is JavaScript?

Imagine your cat, Whiskers, has a favorite cat tower. It's like the structure and style of your web page in HTML and CSS‚Äînice and cozy. But now, think of JavaScript as a magical wand that allows you to add special features to this cat tower.

In the world of cat towers, they're a bit like web pages‚Äîmostly the same. But with JavaScript, it's like sprinkling magic catnip on the tower. It becomes a dynamic playground, responding to Whiskers' movements, adjusting to preferences, and providing an ever-changing environment.

Origins of JavaScript

JavaScript was created in just 10 days in September 1995 by Netscape programmer Brendan Eich. It was originally called Mocha, then LiveScript, before being named JavaScript.

The name was partly a marketing tactic by Netscape to associate it with the popular Java language. However, the two languages are actually very different - Java requires compilation while JavaScript is interpreted.

JavaScript started off being viewed as more of a "UI glue" for designers rather than a serious programming language. But its flexibility and ability to enable interactivity helped it spread rapidly.


Standardization with ECMAScript

By 1997, the growth of JS made it clear there needed to be a standard specification, so Netscape handed it over to ECMA to manage.

This led to the ECMA-262 specification and usage of the term ECMAScript to refer to JS-like languages.

There were revisions to the spec between 1997-1999. After abandoned proposals in the 2000s, ES6 was released in 2015 with many modern features.


Rise in Popularity Due to Ajax

A 2005 paper introduced Ajax, which used JS to enable web apps to feel more responsive without full page reloads.

This helped drive mainstream adoption of JS as a serious development language. Frameworks like jQuery followed to address cross-browser issues.


Node.js Enables Server-Side JS

In 2009, CommonJS specified modules for JS outside browsers, enabling server-side JS.

Node.js provided an environment to run JS code without a browser, allowing JS to be used on servers too.


JavaScript Today

JS now ranks as the most popular programming language globally, with widespread usage across web, mobile, and server-side development.

Modern frameworks like React and Vue allow complex web apps to be built quickly. Some even enable writing native mobile apps entirely in JS.

JavaScript Programs

A computer program is a set of instructions that a computer follows. In programming languages, these instructions are called statements. JavaScript Statements

JavaScript statements are the building blocks of a program and can include values, operators, expressions, keywords, and comments. These statements are executed sequentially, one after the other. Most JavaScript programs consist of numerous statements, forming what is commonly referred to as JavaScript code. Okay, imagine you're writing a set of instructions for your clever cat to follow. Semicolons

Semicolons play the role of separators between JavaScript statements. It's like the punctuation that indicates the end of a programming sentence.   • The iteration statement where you can increase or decrease your counter.

    • You can put all the three parts in a single line separated by semicolons.

Javascript arrays

In JavaScript, an array is a data structure that allows you to store multiple values within a single variable. Arrays are ordered, meaning the position of each element is defined by an index. The index starts at 0 for the first element, 1 for the second, and so on. You can store various types of data, such as numbers, strings, or even other arrays, in an array.

In cat terms... Imagine an array as a collection of different toys for your furry friends. Each toy has its own unique appeal, just like elements in an array. The toys are arranged neatly, and each has a specific spot. For example:

<html>
<body>
<script type = "text/javascript">
// Imagine an array as a collection of your cat's favorite treats.
const catTreats = ["Tuna Bits", "Chicken Bites", "Salmon Delights", "Catnip Cookies"];
// Your cat's treat preferences are indexed, starting from 0.
// Let's check out the first and second treats.
document.write(`${catTreats[0]}` + "<br>"); // Output: Tuna Bits
document.write(`${catTreats[1]}` + "<br>"); // Output: Chicken Bites

// Uh-oh, your cat got a new favorite! Let's update the array.
catTreats[2] = "Shrimp Snacks";

// Great! Now, the array reflects the updated treat collection.
document.write(catTreats + "<br>"); // Output: ["Tuna Bits", "Chicken Bites", "Shrimp Snacks", "Catnip Cookies"]

// Time to share the treats! You can loop through the array and treat your cat to each one.
for (let i = 0; i < catTreats.length; i++) {
document.write(`Time for ${catTreats[i]}!` + "<br>");
}

// Oops, your cat had enough of shrimp. Let's remove it from the array.
catTreats.pop();

// Now, the array is back to three tasty treats.
document.write(catTreats + "<br>"); // Output: ["Tuna Bits", "Chicken Bites", "Catnip Cookies"]
</script>
</body>
