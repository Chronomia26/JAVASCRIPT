What is JavaScript? üêæ

Imagine web pages are like clothing that you wear. You can only do so much with the clothes - they cover you and provide protection, but are mostly static. JavaScript is like adding buttons and zippers to the clothes. Instead of just passively wearing them, you can now interact with them. You can open, zip, and button your clothing to dynamically change how you're protected and uncover different parts as needed.


Similarly, web pages originally were mostly static content, like HTML and CSS that structured and styled the page. But just like adding interactive elements like buttons and zippers to clothes, JavaScript enables you to add dynamic functionality and effects to web pages. You can respond to user actions, detect clicks, change content, validate input, animate objects, load new data‚Äîthings that make web pages feel engaging and responsive.


Zippers and buttons are cheap and lightweight versus extremely fancy and complex mechanisms yet enable so much extra functionality. Similarly, JavaScript code is relatively simple, lightweight and easy to learn, but unlocks tremendous interactivity. Adding a bit of JavaScript code, even to static HTML/CSS sites, brings pages to life just like adding zippers and buttons give plain clothes customizable functionality.

Javascript Statements

JavaScript Programs

A computer program is a set of instructions that a computer follows. In programming languages, these instructions are called statements. A JavaScript program is a collection of these programming statements. When working with HTML, these JavaScript programs get executed by the web browser. Think of it like giving your computer a set of tasks to perform.


JavaScript Statements

JavaScript statements are the building blocks of a program and can include values, operators, expressions, keywords, and comments. These statements are executed sequentially, one after the other. Most JavaScript programs consist of numerous statements, forming what is commonly referred to as JavaScript code. Okay, imagine you're writing a set of instructions for your clever cat to follow. Each instruction is a statement, and the entire set of instructions is a cat code.


Semicolons

Semicolons play the role of separators between JavaScript statements. It's like the punctuation that indicates the end of a programming sentence. While not strictly required, adding a semicolon at the end of each statement is a recommended practice for clarity. Cats, like instructions, need a little pause between tricks. Semicolons are like the treats at the end of each successful trick. While not mandatory, it's a wise idea to reward your cat (or code) with a treat (semicolon) after each task.


JavaScript White Space

JavaScript doesn't fuss about multiple spaces. Adding whitespace to your script can make it more readable. It's akin to giving your code some breathing room. Also, consider putting spaces around operators for better clarity in your instructions. Cats appreciate space‚Äîphysically and in code. Consider white space as the comfy cushions around your cat's favorite perch. It makes the code environment cozy and pleasing to the feline eye.


JavaScript Line Length and Line Breaks

For optimal readability, programmers often avoid code lines longer than 80 characters. If a statement is too long, it's best to break it after an operator. It's like making sure your instructions don't spill over onto the next line without a logical pause. Picture your cat performing an elaborate trick. If the trick is too long, it's best to break it at a logical point, like a graceful pause in a cat's acrobatics. It ensures the audience (or the programmer) can appreciate each part of the performance.


JavaScript Code Blocks

Statements can be grouped together in code blocks, enclosed within curly brackets {...}. This helps define statements that should be executed as a cohesive unit. Imagine these blocks as neatly organized sets of instructions, akin to a well-structured to-do list. Code blocks are like choreographed cat routines. They group together a sequence of tricks that should be executed together. Think of it as organizing your cat's tricks into a delightful routine.


JavaScript Keywords

JavaScript statements often start with keywords that identify the action to be performed. Now, let's add a touch of feline flair to our explanations. Keywords are like magic words that prompt your cat to perform a specific trick. For example, saying "var" is like saying "paw," and your cat knows it's time to declare a variable. These words are the secret language that guides your clever cat through the intricate dance of programming.


So, in the world of programming, you're essentially orchestrating a delightful feline performance where every instruction is a well-crafted trick, and the code blocks are the choreography that makes it all come together seamlessly. Happy coding, or should I say, happy cat-coding! üêæüê±1




JavaScript Datatypes üêæ

One of the most fundamental characteristics of a programming language is the set of data types it supports. These are the type of values that can be represented and manipulated in a programming language. JavaScript allows you to work with three primitive data types ‚àí

  • Numbers, eg. 123, 120.50 etc.
  • Strings of text e.g. "This text is a string" etc.
  • Boolean e.g. true or false.

Note ‚àí JavaScript does not make a distinction between integer values and floating-point values. All numbers in JavaScript are represented as floating-point values. JavaScript represents numbers using the 64-bit floating-point format defined by the IEEE 754 standard.

Comparison Operators

JavaScript supports the following comparison operators ‚àí
Assume a cat named Kitty has 10 toys (represented by "A"), and another cat named Whiskers has 20 toys (represented by "B").

  • Equal Operator (==): Checks if the number of toys Kitty has is equal to the number of toys Whiskers has. if yes, then the condition becomes true.
    Example: (A == B) is not true.

  • Not Equal Operator (!=): Checks if the number of toys Kitty has is not equal to the number of toys Whiskers has. if the values are not equal, then the condition becomes true.
    Example: (A == B) is true.

  • Greater Than Operator (>): Checks if the number of toys Kitty has is greater than the number of toys Whiskers has. if yes, then the condition becomes true.
    Example: (A > B) not is true.

  • Less than Operator (<): Checks if the number of toys Kitty has is less than the number of toys Whiskers has. if yes, then the condition becomes true.
    Example: (A < B) is true.

  • Greater Than Operator (>=): Checks if the number of toys Kitty has is greater than or equal to the number of toys Whiskers has. if yes, then the condition becomes true.
    Example: (A >= B) not is true.

  • Less than Operator (<=): Checks if the number of toys Kitty has is less than or equal to the number of toys Whiskers has. if yes, then the condition becomes true.
    Example: (A <= B) is true.

Logical Operators

Assume that a row (represented by "A") has 10 cats, and another row (represented by "B") has 20 cats.

  • Logical AND (&&) (==): If both rows have cats, the condition is true.
    Example: (A && B) is true.

  • Logical OR (||): If either of the rows have cats, the condition is true.
    Example: (A || B) is true.

  • Logical NOT (!): Reverses the logical state of its operand. If a condition is true, then the Logical NOT operator will make it false.
    Example: !(A && B) is false.

The "for" Loop

The 'for' loop is the most compact form of looping. It includes the following three important parts ‚àí

  • The loop initialization where we initialize our counter to a starting value. The initialization statement is executed before the loop begins.
  • The test statement which will test if a given condition is true or not. If the condition is true, then the code given inside the loop will be executed, otherwise the control will come out of the loop.
  • The iteration statement where you can increase or decrease your counter.

    • You can put all the three parts in a single line separated by semicolons.

Syntax

The syntax for for loop in JavaScript is as follows ‚àí


for (initialization; test condition; iteration statement) {
Statement(s) to be executed if test condition is true
}
